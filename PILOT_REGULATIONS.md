# Регламент пилота ДВИГ (B2C, СПб)

## Цель пилота

Проверить гипотезу «пойти не с кем» на 3–4 недели: событие → заявка → одобрение → офлайн-встреча → check-in → обратная связь.

## География и формат

- Только **Санкт-Петербург** (афиша KudaGo).
- Категории пилота: **Настолки, Кино, Культура** (при необходимости — Спорт).
- Группы **5–7 человек**, только **публичные места**.
- Формат **не 1-на-1** на первом этапе.

## Роли команды

| Роль | Ответственность |
|------|-----------------|
| **Дежурный модератор** | Одобрение/отклонение заявок, создание групп, ответ на жалобы в течение 2 ч |
| **Организатор группы** | Точка встречи, приветствие участников, фиксация check-in |
| **Поле (Полина)** | 5–10 интервью до/после встречи |
| **Разработка** | Техническая поддержка, seed-события, мониторинг API |

## Seed-события

Команда вручную создаёт **5–10 групп** на ближайшие 2 недели через `/admin` или `npm run db:seed`.

Критерии выбора события:

1. Публичная площадка с понятным адресом.
2. Дата в пределах 7–14 дней.
3. Формат, где комфортна группа 5–7 (настолки, кино, выставка, лекция).
4. Возрастное ограничение 18+ или без ограничений.

## Процесс заявки

1. Пользователь регистрируется, заполняет профиль (имя, интересы, город).
2. Выбирает событие с открытой группой → подаёт заявку.
3. Модератор проверяет профиль и одобряет/отклоняет в течение **24 ч**.
4. При одобрении — уведомление (Telegram/email) + точка встречи.
5. В день события — check-in «Я на месте».
6. После события — опрос 1–5 и опциональный комментарий.

## Безопасность

- При **тревожной кнопке**: уведомление дежурному модератору + доверенному контакту пользователя (если указан).
- При **жалобе**: модератор связывается в течение 2 ч, при необходимости — блокировка пользователя.
- Инцидент с угрозой жизни: рекомендовать **112**, зафиксировать в логе модерации.
- Возраст участников: **18+**.

## Критерии успеха

- ≥ **10 check-in** за период пилота.
- ≥ **60%** одобренных заявок доходят до события.
- Медиана «меньше одиноко» T+7 ≥ **3/5**.

## Контакты пилота

Настраиваются через переменные окружения: `NEXT_PUBLIC_PILOT_EMAIL`, `NEXT_PUBLIC_PILOT_TELEGRAM`.
